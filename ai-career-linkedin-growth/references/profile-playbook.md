# LinkedIn Profile Playbook

## Goal

Make the profile immediately understandable for recruiters searching for AI, Python, LLM, RAG, data, MLOps, and agentic AI profiles.

## Profile Strategy

LinkedIn is not just a social network. Treat it as:

- an extension of the CV;
- a professional identity card online;
- a search surface for recruiters;
- a proof hub for projects;
- a networking and watch tool;
- a credibility layer after a recruiter reads the CV.

The profile should answer in 10 seconds:

1. What role is this person targeting?
2. What can they build?
3. What technologies do they use?
4. What proof do they show?
5. Are they available for stage, alternance, first job, or freelance?

Before rewriting the profile, search the user's name online when relevant and flag anything that weakens the professional image. Recommend cleanup or privacy changes for outdated, confusing, or too-personal public results.

Recruiters often use LinkedIn to verify background, understand a profile quickly, and search for specific skills. The profile should therefore be complete, coherent, keyword-rich, and consistent with the CV.

## Visibility Rules

Improve visibility without forcing daily posting:

- complete every useful section because complete profiles are easier to recommend;
- repeat honest keywords in the headline, About, Experience, Education, Skills, and Featured captions;
- grow the network with relevant people: recruiters, HR, CTOs, former colleagues, trainers, clients, classmates, and target-company contacts;
- stay active through likes, comments, shares, and occasional posts;
- follow target companies, sector groups, recruiters, and technical voices for job-market watch.

Quality matters as much as quantity. Avoid sending invitations to many people from the same company at once. Target one or two relevant people and personalize the note.

## Headline Formula

Use:

`Target role | Technical proof | Availability or value`

Examples:

- `Developpeur IA | Python, LLM, RAG, API | Recherche stage 1-3 mois`
- `Developpeur IA Agentique | Agents LLM, RAG, MLOps | Alternance 12 mois`
- `Developpeur Python specialise IA | OpenAI API, FastAPI, PostgreSQL`
- `Developpeur Data & IA | Integration de solutions IA | RAG, APIs, automatisation`
- `Consultant Junior IA | Automatisation, agents LLM, evaluation`

Avoid:

- `Stagiaire IA developer`
- `Dev IA Junior`
- `En formation Developpeur IA agentique a la recherche...`

Reason: vague titles waste search keywords and can sound less credible.

## About Section Structure

Use 4 short blocks.

### Block 1: Positioning

Say the target role and domain.

Example:

`Je me specialise dans le developpement de solutions IA avec Python, les API LLM, les architectures RAG et les workflows agentiques. Je recherche un stage / une alternance / un premier poste en developpement IA.`

### Block 2: Technical reassurance

Show that the user can code.

Mention:

- Python;
- API integration;
- Git/versioning;
- backend basics;
- data handling;
- tests or MLOps if relevant.

### Block 3: AI reassurance

Show that the user is more than a casual GenAI user.

Mention:

- LLM APIs;
- RAG;
- vector databases;
- prompt engineering;
- agent frameworks;
- evaluation;
- monitoring;
- MLOps.

### Block 4: Proof and contact

Mention projects, portfolio, GitHub, and availability.

Example:

`Je documente mes projets sur GitHub et LinkedIn afin de montrer ma progression, mes choix techniques et les resultats obtenus. Ouvert aux opportunites en developpement IA / data & IA / agentic AI.`

## Experience Section

Use this pattern:

`Context -> Mission -> Actions -> Stack -> Result / learning`

For LinkedIn Experience entries, include:

- clear context: mission type, team size, responsibilities;
- detailed tasks and missions;
- technical stack as a readable list;
- role-specific keywords used naturally.

Good bullet template:

- `Concu un prototype de chatbot RAG pour [use case], avec Python, OpenAI API, embeddings et base vectorielle.`
- `Integre une API LLM dans une application backend afin d'automatiser [task].`
- `Mis en place un jeu de tests pour comparer plusieurs versions de prompts et reduire les hallucinations.`
- `Documente les choix techniques, limites et pistes d'amelioration dans le repository GitHub.`

If old experience is not AI-related, translate transferable value:

- client communication;
- problem solving;
- documentation;
- operational reliability;
- teamwork;
- process improvement.

## Education And Certifications

Be precise:

- school;
- diploma/certification;
- dates;
- RNCP level if relevant;
- project-based modules;
- technologies learned;
- real deliverables.

Put high-value certifications near the top when they support the target role.
Mention relevant tech certifications even when they are time-limited or no longer active, but do not imply they are current if they expired. Order the section so the strongest proof appears first.

## Skills

Organize by category.

Recommended categories:

- Programming: Python, JavaScript, SQL
- AI/LLM: OpenAI API, Anthropic API, embeddings, RAG, prompt engineering
- Agentic AI: agents, tool calling, orchestration, evaluation
- Backend/API: FastAPI, REST APIs, authentication basics
- Data: PostgreSQL, vector databases, data cleaning
- MLOps: Git, versioning, evaluation, monitoring, CI/CD basics
- Soft skills: communication, documentation, autonomy, teamwork

## Featured Section

Pin proof:

1. GitHub portfolio repository.
2. Best AI project.
3. LinkedIn post explaining a project.
4. CV PDF.
5. Training flyer, certificate, or formation document if useful.

Each featured item should have a short caption:

`Projet RAG: chatbot documentaire avec Python, embeddings, base vectorielle et evaluation des reponses.`

Use the Featured section to make proof visible before a recruiter has to search for it. Good items are a CV, GitHub project, portfolio page, formation flyer, certificate, project demo, or strong LinkedIn post.

## Recommendations

Ask for specific recommendations from:

- former colleagues;
- trainers;
- project teammates;
- clients;
- internship supervisors.

Message template:

`Bonjour [Name], je travaille actuellement mon profil LinkedIn pour ma recherche de [stage/alternance/poste] en developpement IA. Si tu es d'accord, pourrais-tu me laisser une courte recommandation sur [collaboration/projet/competence precise] ? Je peux aussi te proposer quelques points pour te faciliter la redaction.`

## Visual Advice

- Use a clear, professional, face-focused photo.
- Avoid casual photos that do not signal seriousness.
- Avoid overly formal ID-card style photos when they make the profile feel rigid.
- Use an OpenToWork frame when active search visibility is more useful than discretion.
- Use a banner that signals AI/data/engineering without looking too generic.
- Use LinkedIn banner dimensions `1584 x 396 px`.
- Use special text styles sparingly.
- Keep profile sections consistent with the CV.

## Connection Note Template

Keep invitation notes under 300 characters.

Template:

`Bonjour [Name], j'ai vu votre travail sur [topic/company/event]. Je construis mon projet vers [target role] et vos contenus sont alignes avec ma veille. Au plaisir d'echanger ou de suivre vos actualites.`

## Profile Benchmark Exercise

When the user lacks keywords or examples, ask them to inspect 3 to 5 profiles similar to the target role and extract:

- repeated job titles;
- recurring technologies;
- certifications;
- project types;
- headline patterns;
- skills and wording that feel credible.
