# Playbook Profil LinkedIn

## Objectif

Rendre le profil immédiatement compréhensible pour les recruteurs qui cherchent des profils IA, Python, LLM, RAG, data, MLOps et IA agentique.

## Stratégie Profil

LinkedIn n'est pas seulement un réseau social. Le traiter comme :

- une extension du CV ;
- une carte d'identité professionnelle en ligne ;
- une surface de recherche pour les recruteurs ;
- un hub de preuves projet ;
- un outil de réseau et de veille ;
- une couche de crédibilité après lecture du CV.

Le profil doit répondre en 10 secondes :

1. Quel rôle cette personne vise-t-elle ?
2. Que peut-elle construire ?
3. Quelles technologies utilise-t-elle ?
4. Quelles preuves montre-t-elle ?
5. Est-elle disponible pour un stage, une alternance, un premier emploi ou du freelance ?

Avant de réécrire le profil, rechercher le nom de l'utilisateur en ligne quand c'est pertinent et signaler tout élément qui affaiblit l'image professionnelle. Recommander nettoyage ou réglages de confidentialité pour les résultats publics obsolètes, confus ou trop personnels.

Les recruteurs utilisent souvent LinkedIn pour vérifier un parcours, comprendre rapidement un profil et chercher des compétences précises. Le profil doit donc être complet, cohérent, riche en mots-clés et aligné avec le CV.

Si l'utilisateur a déjà fait un scan CV, réutiliser les preuves vérifiées et le titre cible de ce scan. Ne pas ajouter sur LinkedIn des mots-clés que l'analyse CV a marqués comme risqués ou non vérifiés.

## Règles De Visibilité

Améliorer la visibilité sans forcer la publication quotidienne :

- compléter toutes les sections utiles car les profils complets sont plus faciles à recommander ;
- répéter les mots-clés honnêtes dans le titre, le À propos, les Expériences, la Formation, les Compétences et les légendes de la section Sélection ;
- développer le réseau avec des personnes pertinentes : recruteurs, RH, CTO, anciens collègues, formateurs, clients, camarades et contacts d'entreprises cibles ;
- rester actif avec likes, commentaires, partages et posts occasionnels ;
- suivre entreprises cibles, groupes du secteur, recruteurs et voix techniques pour la veille.

La qualité compte autant que la quantité. Éviter d'envoyer beaucoup d'invitations à plusieurs personnes d'une même entreprise en même temps. Cibler une ou deux personnes pertinentes et personnaliser la note.

## Formule De Titre

Utiliser :

`Rôle cible | Preuve technique | Disponibilité ou valeur`

Exemples :

- `Développeur IA | Python, LLM, RAG, API | Recherche stage 1-3 mois`
- `Développeur IA Agentique | Agents LLM, RAG, MLOps | Alternance 12 mois`
- `Développeur Python spécialisé IA | OpenAI API, FastAPI, PostgreSQL`
- `Développeur Data & IA | Intégration de solutions IA | RAG, API, automatisation`
- `Consultant Junior IA | Automatisation, agents LLM, évaluation`

Éviter :

- `Stagiaire IA developer`
- `Dev IA Junior`
- `En formation Développeur IA agentique à la recherche...`

Raison : les titres vagues gaspillent les mots-clés de recherche et peuvent sembler moins crédibles.

## Structure Du À Propos

Utiliser 4 blocs courts.

### Bloc 1 : Positionnement

Dire le rôle cible et le domaine.

Exemple :

`Je me spécialise dans le développement de solutions IA avec Python, les API LLM, les architectures RAG et les workflows agentiques. Je recherche un stage / une alternance / un premier poste en développement IA.`

### Bloc 2 : Réassurance Technique

Montrer que l'utilisateur sait coder.

Mentionner :

- Python ;
- intégration API ;
- Git/versionnement ;
- bases côté serveur ;
- traitement de données ;
- tests ou MLOps si pertinent.

### Bloc 3 : Réassurance IA

Montrer que l'utilisateur dépasse la simple utilisation d'outils GenAI.

Mentionner :

- API LLM ;
- RAG ;
- bases vectorielles ;
- prompt engineering ;
- frameworks d'agents ;
- évaluation ;
- surveillance ;
- MLOps.

### Bloc 4 : Preuve Et Contact

Mentionner projets, portfolio, GitHub et disponibilité.

Exemple :

`Je documente mes projets sur GitHub et LinkedIn afin de montrer ma progression, mes choix techniques et les résultats obtenus. Ouvert aux opportunités en développement IA / data & IA / IA agentique.`

## Section Expérience

Utiliser ce modèle :

`Contexte -> Mission -> Actions -> Technologies -> Résultat / apprentissage`

Pour les expériences LinkedIn, inclure :

- contexte clair : type de mission, équipe, responsabilités ;
- tâches et missions détaillées ;
- technologies lisibles ;
- mots-clés du rôle utilisés naturellement.

Bon modèle de puce :

- `Conçu un prototype de chatbot RAG pour [cas d'usage], avec Python, OpenAI API, embeddings et base vectorielle.`
- `Intégré une API LLM dans une application côté serveur afin d'automatiser [tâche].`
- `Mis en place un jeu de tests pour comparer plusieurs versions de prompts et réduire les hallucinations.`
- `Documenté les choix techniques, limites et pistes d'amélioration dans le dépôt GitHub.`

Si l'ancienne expérience n'est pas liée à l'IA, traduire la valeur transférable :

- communication client ;
- résolution de problèmes ;
- documentation ;
- fiabilité opérationnelle ;
- travail d'équipe ;
- amélioration de processus.

## Formation Et Certifications

Être précis :

- école ;
- diplôme/certification ;
- dates ;
- niveau RNCP si pertinent ;
- modules par projets ;
- technologies apprises ;
- livrables réels.

Placer les certifications à forte valeur près du haut quand elles soutiennent le rôle cible. Mentionner les certifications techniques pertinentes même si elles sont limitées dans le temps ou expirées, mais ne pas les présenter comme actuelles si elles ne le sont plus.

## Compétences

Organiser par catégorie.

Catégories recommandées :

- Programmation : Python, JavaScript, SQL
- IA/LLM : OpenAI API, Anthropic API, embeddings, RAG, prompt engineering
- IA Agentique : agents, appel d'outils, orchestration, évaluation
- Backend/API : FastAPI, API REST, bases d'authentification
- Données : PostgreSQL, bases vectorielles, nettoyage de données
- MLOps : Git, versionnement, évaluation, surveillance, bases CI/CD
- Compétences comportementales : communication, documentation, autonomie, travail d'équipe

## Section Sélection

Épingler les preuves :

1. Dépôt GitHub portfolio.
2. Meilleur projet IA.
3. Post LinkedIn expliquant un projet.
4. CV PDF.
5. Plaquette de formation, certificat ou document de formation si utile.

Chaque élément sélectionné doit avoir une courte légende :

`Projet RAG : chatbot documentaire avec Python, embeddings, base vectorielle et évaluation des réponses.`

Utiliser la section Sélection pour rendre les preuves visibles avant qu'un recruteur ait besoin de chercher. Bons éléments : CV, projet GitHub, portfolio, plaquette de formation, certificat, démo projet ou post LinkedIn fort.

Lors de l'alignement avec un CV réécrit, placer d'abord les preuves les plus fortes du CV : meilleur projet IA, dépôt GitHub, CV PDF et post technique expliquant le projet.

## Recommandations

Demander des recommandations précises à :

- anciens collègues ;
- formateurs ;
- coéquipiers de projet ;
- clients ;
- responsables de stage.

Modèle de message :

`Bonjour [Nom], je travaille actuellement mon profil LinkedIn pour ma recherche de [stage/alternance/poste] en développement IA. Si tu es d'accord, pourrais-tu me laisser une courte recommandation sur [collaboration/projet/compétence précise] ? Je peux aussi te proposer quelques points pour te faciliter la rédaction.`

## Conseils Visuels

- Utiliser une photo claire, professionnelle et centrée sur le visage.
- Éviter les photos trop casual qui ne signalent pas le sérieux.
- Éviter la photo d'identité trop rigide si elle donne une impression froide.
- Utiliser le cadre OpenToWork quand la visibilité active est plus utile que la discrétion.
- Utiliser une bannière qui signale IA/data/engineering sans paraître trop générique.
- Utiliser les dimensions LinkedIn `1584 x 396 px`.
- Utiliser les styles de texte spéciaux avec parcimonie.
- Garder les sections cohérentes avec le CV.

## Modèle De Note De Connexion

Garder les notes d'invitation sous 300 caractères.

Modèle :

`Bonjour [Nom], j'ai vu votre travail sur [sujet/entreprise/événement]. Je construis mon projet vers [rôle cible] et vos contenus sont alignés avec ma veille. Au plaisir d'échanger ou de suivre vos actualités.`

## Exercice De Benchmark Profil

Quand l'utilisateur manque de mots-clés ou d'exemples, lui demander d'inspecter 3 à 5 profils similaires au rôle cible et d'extraire :

- titres de poste répétés ;
- technologies récurrentes ;
- certifications ;
- types de projets ;
- modèles de titres ;
- compétences et formulations crédibles.
