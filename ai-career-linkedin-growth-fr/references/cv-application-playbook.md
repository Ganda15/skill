# Playbook CV Et Candidature

## Stratégie CV

Le CV doit être lisible par un ATS et par un recruteur.

Pour une analyse, une comparaison avec une offre ou une réécriture complète, charger d'abord `cv-power-scan.md`, puis utiliser ce fichier pour les formulations finales, les supports de candidature et la routine de recherche.

Quand une offre est fournie, ne pas écrire la lettre ou le message recruteur avant d'avoir extrait le titre du poste, les compétences demandées, les responsabilités et les besoins de l'entreprise.

Priorités :

1. Un rôle cible clair.
2. Les mots-clés pertinents de l'offre.
3. Des preuves par les projets et résultats.
4. Une stack technique groupée clairement.
5. Aucune revendication gonflée.

## Structure CV

Ordre recommandé pour un Développeur IA junior :

1. Nom + titre + contact + LinkedIn + GitHub.
2. Résumé professionnel, 3 à 5 lignes.
3. Compétences techniques groupées par catégorie.
4. Projets IA / logiciels.
5. Expérience.
6. Formation.
7. Certifications.
8. Langues.

Si l'utilisateur a peu d'expérience IA, placer les projets avant les anciens emplois.

Pour les rôles Développeur IA / IA Agentique, privilégier les preuves projet avant les expériences sans lien direct quand les projets prouvent mieux la compétence.

## Formule De Résumé

Utiliser :

`Rôle cible + stack technique + preuve + disponibilité`

Exemple :

`Développeur IA en formation, spécialisé dans l'intégration de solutions LLM avec Python, API, RAG et bases de données. Expérience projet sur chatbots, évaluation d'agents et workflows MLOps simples. Recherche un stage de 1 à 3 mois pour contribuer à des cas d'usage IA concrets.`

## Formule De Puce Projet

Utiliser :

`Développé / conçu / intégré + objet + stack + objectif + résultat ou apprentissage`

Exemples :

- `Développé un prototype de chatbot RAG avec Python, OpenAI API, embeddings et base vectorielle pour interroger une documentation projet.`
- `Créé une fiche d'évaluation d'agent IA avec jeux de cas, métriques et tests automatisables afin de comparer plusieurs versions.`
- `Modélisé une boucle MLOps minimale versionner -> livrer -> suivre pour contrôler la qualité d'un agent LLM.`
- `Intégré une API LLM dans un flux Python pour automatiser la génération et l'analyse de réponses structurées.`

## Formule De Puce Expérience

Utiliser :

`Contexte + action + outil/méthode + résultat`

Éviter :

- uniquement les responsabilités ;
- les longs paragraphes ;
- les verbes faibles comme "participé" ;
- les chiffres non vérifiés.

Préférer :

- `Analysé`
- `Développé`
- `Intégré`
- `Automatisé`
- `Documenté`
- `Testé`
- `Comparé`
- `Amélioré`

## Structure De Lettre De Motivation

Rester sous 400 mots sauf demande contraire.

Utiliser 4 paragraphes :

1. Pourquoi ce rôle ou cette entreprise.
2. Pourquoi l'utilisateur correspond au rôle.
3. Preuve : un ou deux projets pertinents.
4. Disponibilité et appel à l'échange.

Modèle :

`Bonjour,`

`Votre offre pour [rôle] m'intéresse car elle correspond à mon objectif : contribuer à des solutions IA concrètes, utiles et bien intégrées dans un produit ou un processus métier.`

`Je me forme au développement IA avec une base technique en Python, intégration API, Git, bases de données, RAG, prompt engineering et évaluation d'agents. Mes projets m'ont appris à construire, tester et documenter des solutions basées sur des LLM, au-delà d'une simple utilisation d'outils GenAI.`

`Par exemple, j'ai travaillé sur [projet], où j'ai [action] avec [stack] afin de [résultat/objectif]. Ce type de mission m'intéresse car il relie développement, expérimentation et qualité.`

`Je serais ravi d'échanger avec vous sur le poste et sur la manière dont je pourrais contribuer à vos projets IA.`

## Message Recruteur

Rester court et spécifique.

Modèle :

`Bonjour [Nom], je vous contacte car je recherche [stage/alternance/poste] en développement IA / IA Agentique. Je travaille sur Python, API LLM, RAG, évaluation d'agents et MLOps minimal. J'ai vu que [entreprise] travaille sur [sujet]. Seriez-vous ouvert à un court échange ou à recevoir mon CV ?`

Si une offre est incluse, mentionner une exigence de l'offre et une preuve du CV. Rester honnête et ne pas répéter toute la lettre.

## Note De Connexion LinkedIn

Quand le but est seulement de se connecter, rester sous 300 caractères et contextualiser.

Modèle :

`Bonjour [Nom], je m'intéresse à [entreprise/sujet] dans le cadre de ma recherche en [rôle cible]. Votre parcours/contenu est très utile pour ma veille. Je serais ravi de vous suivre ici.`

Ne pas envoyer la même note générique à beaucoup de personnes d'une même entreprise. Choisir d'abord une ou deux personnes pertinentes.

## Message De Relance

Modèle :

`Bonjour [Nom], je me permets de revenir vers vous concernant ma candidature pour [rôle]. Je reste très intéressé, notamment par [point spécifique]. Je peux vous transmettre un projet GitHub ou une synthèse de mes travaux IA si utile. Merci pour votre retour.`

## Routine De Recherche D'Emploi

Routine hebdomadaire :

- Lundi : mettre à jour la liste cible et candidater à 5 offres de qualité.
- Mardi : contacter 5 personnes dans les entreprises cibles.
- Mercredi : publier ou commenter du contenu IA / IA agentique.
- Jeudi : adapter CV et lettre pour les meilleures offres.
- Vendredi : analyser les retours et améliorer les mots-clés du profil.

Suivre :

- titre du poste ;
- entreprise ;
- source ;
- date de candidature ;
- personne contactée ;
- statut ;
- prochaine action.

Ajouter des actions de veille LinkedIn :

- suivre entreprises et groupes cibles ;
- suivre recruteurs, RH, CTO et profils techniques ;
- collecter les mots-clés récurrents de 3 à 5 profils similaires ;
- identifier une personne par entreprise cible avant d'envoyer une note de connexion.
