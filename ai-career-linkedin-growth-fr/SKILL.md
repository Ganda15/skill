# Croissance Carrière IA Et LinkedIn

## Vue D'Ensemble

Utiliser cette compétence pour transformer un profil apprenant, junior ou en reconversion en système de recherche d'emploi crédible : CV, profil LinkedIn, projets visibles, posts, lettres de motivation, messages recruteurs, preuves de compétences et plan d'action.

Principe central :

> Être trouvable, crédible, précis et fondé sur des preuves. Ne pas sonner générique, désespéré ou artificiellement marketing.

## Flux De Travail

1. Identifier la cible : stage, alternance, premier emploi, CDI junior, freelance ou visibilité portfolio.
2. Extraire les preuves de l'utilisateur : projets, stack, formation, certifications, expériences, contraintes, langue, localisation, disponibilité.
3. Construire le positionnement : un titre professionnel clair, une famille de rôles cible, une narration basée sur les preuves.
4. Optimiser LinkedIn : image en ligne, titre, À propos, Expériences, Formation, Compétences, Sélection, Recommandations, réseau, routine de veille.
5. Pour les demandes CV, faire d'abord un scan : rôle cible, preuves actuelles, mots-clés IA/agentique, risques ATS, compétences manquantes, puces faibles.
6. Si une offre d'emploi est fournie, comparer le CV avec l'offre avant de réécrire.
7. Optimiser le CV : structure compatible ATS, mots-clés, puces quantifiées quand c'est vrai, stack technique, projets.
8. Si l'utilisateur demande une "option puissante", utiliser une compétence CV/ATS dédiée ou une recherche de compétences comme second passage de contrôle.
9. Créer les supports de candidature : lettre de motivation, message recruteur, relance, note de connexion.
10. Créer une stratégie de contenu : posts LinkedIn qui montrent apprentissage, projets, jugement technique et voix humaine.
11. Appliquer la liste de contrôle avant de finaliser.

## Informations À Demander

Demander seulement les informations manquantes qui changent vraiment le résultat :

- rôle et contrat cible : stage, alternance, CDI, freelance ;
- localisation et disponibilité ;
- CV actuel ou texte LinkedIn ;
- offre d'emploi si la candidature doit être adaptée ;
- choix souhaité : scan rapide, réécriture complète, comparaison avec une offre ;
- besoin éventuel : alignement LinkedIn, pack de candidature, second passage ATS ;
- projets avec stack, objectif, résultat, liens, captures, démos ;
- formation, certifications, école, dates ;
- langues et autorisation de travail si pertinent ;
- ton préféré : français, anglais, bilingue, direct, chaleureux, technique.

## Standards De Sortie

Produire des sorties pratiques que l'utilisateur peut coller ou publier :

- 3 à 7 options de titre LinkedIn ;
- diagnostic CV avec forces, risques, preuves manquantes, mots-clés manquants et prochaines actions ;
- tableau de correspondance CV/offre quand une offre est fournie ;
- proposition de CV réécrit pour Développeur IA ou Développeur IA Agentique ;
- plan de compétences manquantes regroupé en apprendre, prouver, ajouter au CV quand vérifié ;
- section À propos LinkedIn polie ;
- puces d'expérience avec contexte + action + outils + résultat ;
- liste de compétences organisée par catégorie ;
- recommandations pour la section Sélection ;
- notes de connexion LinkedIn sous 300 caractères ;
- lettre de motivation sous 400 mots sauf demande contraire ;
- messages recruteurs/réseau sous 900 caractères ;
- posts LinkedIn avec accroche, preuve, leçon et appel doux à l'échange ;
- plan hebdomadaire de recherche d'emploi si demandé.

## Règles De Positionnement

Préférer les titres précis aux étiquettes vagues.

Éviter :

- "Stagiaire IA developer"
- "Dev IA junior"
- "En formation développeur IA agentique à la recherche..."

Préférer :

- "Développeur IA | Python, LLM, RAG, API"
- "Développeur IA Agentique | Recherche alternance 12 mois"
- "Développeur Python spécialisé IA | OpenAI API, RAG, MLOps"
- "Développeur Data & IA | Intégration de solutions IA"
- "Consultant Junior IA | Automatisation, RAG, agents LLM"

Pour les profils en reconversion, rendre la transition claire mais non dominante. Commencer par le rôle cible et les preuves, puis mentionner la formation.

## Règles De Contenu LinkedIn

Pour les posts, optimiser l'authenticité et l'utilité :

- écrire à partir d'un vrai projet ou apprentissage ;
- montrer le problème, la décision, le résultat et la leçon ;
- éviter les posts génériques du type "l'IA change tout" ;
- éviter les appels artificiels à l'engagement ;
- éviter le ton trop modèle ;
- utiliser l'IA pour affiner la voix de l'utilisateur, pas pour la remplacer.

Bons thèmes de posts IA Agentique :

- "Ce que j'ai appris en construisant un chatbot RAG"
- "Comment j'évalue un agent IA avant déploiement"
- "Une boucle MLOps simple pour agents IA"
- "Une erreur que j'ai faite avec une API LLM"
- "Comment j'ai débogué des hallucinations dans un chatbot"
- "Ce que signifie IA agentique en pratique, sans hype"

## Règles De Visibilité Recruteur

Utiliser les mots-clés à plusieurs endroits, pas seulement dans les Compétences :

- titre LinkedIn ;
- trois premières lignes du À propos ;
- puces d'expérience ;
- projets ;
- description de formation ;
- compétences ;
- légendes de la section Sélection.

Les mots-clés doivent rester honnêtes. Ne pas revendiquer une expérience production, un déploiement ou une expertise sans preuve.

## Barre Qualité

Avant la réponse finale, vérifier :

- Le titre indique-t-il clairement le rôle cible ?
- Le À propos rassure-t-il sur la partie code et sur la partie IA ?
- L'analyse CV sépare-t-elle les preuves vérifiées des améliorations suggérées ?
- Si une offre est fournie, les mots-clés correspondants et les exigences manquantes sont-ils explicites ?
- Les projets sont-ils assez concrets pour prouver la capacité ?
- Les technologies sont-elles groupées logiquement ?
- Les puces d'expérience sont-elles orientées action ?
- Le CV est-il compatible ATS ?
- Le post LinkedIn sonne-t-il humain ?
- La prochaine action est-elle claire pour l'utilisateur ?

Si l'utilisateur demande une publication GitHub, garder le dossier de compétence propre. Ne pas ajouter de documentation sans lien direct. Vérifier `SKILL.md`, `agents/openai.yaml` et les références.
